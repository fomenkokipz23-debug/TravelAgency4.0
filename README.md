#  TravelAgency.Web - Онлайн-Конструктор Подорожей

**TravelAgency.Web** - це сучасний вебзастосунок для автоматизації процесів туристичного агентства, розроблений на платформі **.NET 10** з використанням **ASP.NET Core MVC**. 

Головна особливість системи - інтерактивний **онлайн-конструктор**, який дозволяє клієнту гнучко кастомізувати свій відпочинок (обирати клас перельоту, зірковість готелю, додавати екскурсії) з миттєвим динамічним перерахунком вартості.

---

##  Структура Проєкту (Solution)

Проєкт спроєктовано з розподілом на три логічні шари (модулі) відповідно до архітектурних принципів розділення обов'язків:

1. **`TravelAgency.Domain` (Ядро бізнес-логіки):**
   * Містить доменні моделі, об'єкти-значення, кастомні винятки, сервіси управління та інтерфейси. Повністю ізольований від презентаційного шару та баз даних.
2. **`TravelAgency.Web` (Презентаційний шар):**
   * Веб-інтерфейс на базі ASP.NET Core MVC. Включає контролери (`HomeController`, `AccountController`), моделі представлення (DTO) та адаптивний фронтенд (Bootstrap 5, Razor, JavaScript).
3. **`TravelAgency.Tests` (Модуль тестування):**
   * Набір автоматизованих юніт-тестів на базі фреймворку **xUnit** для покриття критичної бізнес-логіки та патернів.

##  Патерни Проєктування та SOLID

Архітектура коду демонструє глибоке застосування об'єктно-орієнтованих принципів та класичних патернів GoF:

* **Singleton (Одинак):** Реалізовано в `AgencyRegistry` для створення єдиної, потокобезпечної (`lock`) точки доступу до сервісів додатка протягом усього сеансу.
* **Builder (Будівельник):** Реалізовано через `ComplexTourBuilder`. Забезпечує покрокове та гнучке конструювання об'єкта `CustomTour` через ланцюжок методів (Fluent Interface).
* **Strategy (Стратегія):** Впроваджено через сімейство класів `IPriceStrategy`. Дозволяє системі динамічно підміняти алгоритми розрахунку цін (акційні знижки `-20%` або сезонні націнки `+15%`) без зміни логіки доменних моделей (**Принцип OCP**).
* **Observer (Спостерігач):** Реалізовано за допомогою стандартних .NET подій (`event EventHandler`). При оформленні туру сервіс автоматично тригерить подію, яку асинхронно обробляє логер `NotificationLogService`.
* **Принцип DIP (Інверсія залежностей):** Контролери веб-додатка залежать виключно від абстракцій (`ITourReader`), які автоматично впроваджуються через вбудований IoC-контейнер (.NET Dependency Injection).

##  Ключові Технології та Фічі

* **Generics (Узагальнення):** Створено універсальний, типобезпечний репозиторій `Repository<T>` з обмеженнями параметрів типу (`where T : class`).
* **LINQ & Лямбда-вирази:** Використовуються для високорівневої фільтрації, проекції та фінансової агрегації даних (наприклад, формування індивідуальної історії подорожей користувача за його Email).
* **Безпека та Сесії:** Реалізовано автентифікацію користувачів на основі Cookie (`.NET Cookie Authentication`) та безпечний розподіл прав через атрибути `[Authorize]`.
* **Обробка винятків:** Створено кастомні типи помилок бізнес-логіки (`InvalidRouteException`, `TravelServiceException`) та впроваджено стійку політику повторних спроб (**Retry Policy**) з експоненційним затриманням часу.
* **Перевантаження операторів:** У класі `Route` перевантажено оператор `+`, що дозволяє лаконічно збирати маршрути: `route = route + "Київ" + "Львів" + "Париж"`.
* **Серіалізація:** Інтегровано `System.Text.Json` для маппінгу та збереження стану замовлень у людиночитабельні файли-бекапи (`booking_backup.json`).
